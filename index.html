Ambiente gráfico é o nome dado à interface projetada para facilitar a utilização do computador pelo usuário a partir de representações visuais do sistema operacional, de elementos gráficos e indicadores visuais.
Interface é por definição, o conjunto de recursos que permitem a um sistema adaptar-se a outro, seja ele físico ou virtual.

Do ponto de vista do usuário de TI, existem dois tipos de interfaces: GUI (Graphic User Interface) e CLI (Command Line Interface)․

A GUI é composta por todos os recursos gráficos utilizados pelo usuário para que o mesmo consiga executar suas tarefas facilmente, sem a necessidade de conhecimento profundo de comandos. O cursor piscando intermitentemente em nossa tela, os ícones em nosso desktop dos aplicativos nativos ou instalados em nosso sistema, botões de acão, links e o próprio assistente de instalação de programas são alguns exemplos de elementos da Interface Gráfica do Usuário․

A CLI por sua vez é destinada aos usuários avançados do sistema e que necessitam executar tarefas não acessíveis, ou não disponíveis no modo GUI. O emulador de terminal para o Ambiente de Trabalho MATE no Linux Mint e o CMD do Windows são exemplos de interfaces que utilizam linha de comando․

Agora que sabemos o que é o GitHub, precisamos entender que é o Git. O Git é um sistema de controle de versão desenvolvido por Linus Torvalds (o criador do Linux).

Sempre que desenvolvedores criam um novo projeto, são necessárias atualizações no código base, afim de permitir atualização de versões, correção de bugs, adição de novas ferramentas, etc. O sistema de controle de versão ajuda a acompanhar as mudanças feitas no código base. E mais, ele também registra quem efetuou a mudança e permite a restauração do código removido ou modificado. O Emmet é um plug-in para muitos editores de texto populares, que melhora a produtividade e aprimora bastante o fluxo de trabalho de desenvolvedores HTML e CSS. É escrito em JavaScript puro e funciona em diferentes plataformas: navegador da web, Node.js, Microsoft WSH e Mozilla Rhino. Um kit de ferramentas essencial para desenvolvedores da web

Lúcio Lemos Postado em Dezembro 13, 2019

A maioria dos editores de texto disponíveis permite armazenar e reutilizar trechos de código que se repetem no código-fonte, chamados de ″snippets″. Embora os snippets sejam uma boa maneira de aumentar sua produtividade, todas as implementações têm armadilhas comuns: você precisa definir os snippets primeiro e não pode estendê-los em tempo de execução.

O Emmet leva a idéia de snippets a um nível totalmente novo: você pode digitar expressões semelhantes a CSS que podem ser analisadas dinamicamente e produzir saída dependendo do que você digitar na abreviação. O Emmet é desenvolvido e otimizado para desenvolvedores da Web cujo fluxo de trabalho depende de HTML / XML e CSS, mas também pode ser usado com linguagens de programação.

Abreviações são o coração do kit de ferramentas Emmet: essas expressões especiais são analisadas em tempo de execução e transformadas em bloco de código estruturado, HTML por exemplo. A sintaxe da abreviação se parece com seletores CSS com algumas extensões específicas para geração de código. Suas principais características são:

  • É gratuito e de código aberto
  • Sintaxe inspirada nos seletores CSS;
  • Escrito em javascript;
  • Codificação ultra-rápida;
  • …

Caminho do arquivo de configuração do apache2 no Linux Mint: etc/apache2/apache2.conf.

  1. Verificando a versão do Servidor Apache HTTP no terminal do Apache Netbeans via interface de linha de comando(modo CLI):
    luciolemos@dell:~$ apache2 -v
    Server version: Apache/2.4.29 (Ubuntu)
    Server built: 2019-09-16T12:58:48

  2. Verificando o status do Servidor Apache HTTP de duas maneiras distintas:
    luciolemos@dell:~$ systemctl status apache2

  3. Iniciando o Servidor Apache HTTP de duas maneiras distintas:
    luciolemos@dell:~$ systemctl start apache2

  4. Parando o Servidor Apache HTTP de duas maneiras distintas:
    luciolemos@dell:~$ systemctl stop apache2
    luciolemos@dell:~$ /etc/init.d/apache2 stop

MySQL Community Server - Tips de comandos no terminal

  1. Verificando a versão do servidor de banco de dados instalada:
    luciolemos@dell:~$ mysql -V
    mysql Ver 14.14 Distrib 5.7.28, for Linux (x86_64) using EditLine wrapper
    luciolemos@dell:~$
  2. Verificando o status do Servidor de Banco de Dados MySQL Community Server:
    luciolemos@dell:~$ /etc/init.d/mysql status
  3. Inicializando o Servidor de Banco de Dados MySQL Community Server. É necessário autenticação como usuário root do SO (password: root), para iniciar o mysql.service:
    luciolemos@dell:~$ /etc/init.d/mysql start
    [ ok ] Starting mysql (via systemctl): mysql.service.
    luciolemos@dell:~$
  4. Após inicializar o Servidor de Banco de Dados MySQL Community Server com autenticação root, querendo acessar a base de dados no servidor, via terminal, execute como usuário root (password: Lemos35@) do SGBD:
    luciolemos@dell:~$ mysql -u root -p
    Enter password: ********
    mysql>
  5. Iniciando o SGBD MySql Workbech Community 6.3 via terminal:
    luciolemos@dell:~$ /usr/bin/mysql-workbench
  6. Finalizando o Servidor de Banco de Dados MySQL Community Server:
    luciolemos@dell:~$ /etc/init.d/mysql stop
    [ ok ] Stopping mysql (via systemctl): mysql.service.
    luciolemos@dell:~$

O PHP, acrônimo recursivo para "Hypertext Preprocessor" é uma linguagem de script open source de uso geral, muito utilizada, e especialmente adequada para o desenvolvimento web e que pode ser embutida dentro do HTML.

Lúcio Lemos Postado em Dezembro 13, 2019

Criado por Rasmus Lerdorf em 1995, com o nome Personal Home Page Tools, o PHP tem a produção de sua implementação principal, referência formal da linguagem, mantida por uma organização chamada The PHP Group. O PHP é software livre, licenciado sob a PHP License, uma licença incompatível com a GNU General Public License (GPL) devido a restrições no uso do termo PHP.

O que distingue o PHP de algo como o JavaScript no lado do cliente é que o código é executado no servidor, gerando o HTML que é então enviado para o navegador. O navegador recebe os resultados da execução desse script, mas não sabe qual era o código fonte. Você pode inclusive configurar seu servidor web para processar todos os seus arquivos HTML com o PHP, e então não há como os usuários dizerem o que você tem na sua manga.

O PHP pode ser utilizado na maioria dos sistemas operacionais, incluindo Linux, várias variantes do Unix (como HP-UX, Solaris e OpenBSD), Microsoft Windows, Mac OS X, RISC OS e provavelmente outros. O PHP também tem suporte à maioria dos servidores web atualmente e isso inclui o Apache

Todos os dias, a internet gera 2,5 quintilhões de bytes de dados. Ao passar seu cartão de crédito, navegar por um website, clicar em um anúncio ou navegar nas mídias sociais, você adiciona ao pool de dados. A ciência de dados, a arte de dar sentido a tudo isso, é um campo de trabalho emergente que muitas empresas de tecnologia e líderes de pensamento consideram um dos mais importantes agora e no futuro.
